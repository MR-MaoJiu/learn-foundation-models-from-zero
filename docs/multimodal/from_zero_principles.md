# 多模态模型从 0 原理

多模态模型的目标是让不同类型的数据可以互相理解。

这个项目先做一个非常小的图文对齐模型 TinyCLIP。它学习的问题是：

```text
这张图片和这句话是不是一对？
```

我们用 MNIST 图片和中文描述构造最小可理解的数据：

```text
图片：一张手写数字 7
文本：这是一张手写数字七。
```

这不是工业级多模态模型，但它包含了 CLIP 类模型的核心思想。

## 1. 什么是“对齐”

LLM 只处理文本，视觉模型只处理图片。多模态模型需要把它们放到同一个向量空间里。

所谓向量空间，可以先理解成一个坐标系统。

模型会把图片变成一个向量，也会把文本变成一个向量：

```text
图片 -> image_vector
文本 -> text_vector
```

如果图片和文本匹配，这两个向量应该接近。

如果不匹配，它们应该远离。

## 2. 为什么要输出向量，而不是直接分类

视觉分类模型输出的是：

```text
[数字0分数, 数字1分数, ..., 数字9分数]
```

它只能回答固定类别问题。

多模态对齐模型输出向量后，可以做更多事情：

- 给一张图片，找最匹配的文字。
- 给一句文字，找最匹配的图片。
- 把图片和文字作为同一种“语义表示”参与后续任务。

这就是多模态模型比单纯分类模型更通用的地方。

## 3. 图片编码器做什么

图片编码器负责：

```text
图片张量 [batch, 1, 28, 28] -> 图片向量 [batch, embed_dim]
```

在本项目里，图片编码器是一个小 CNN。

它和视觉分类模型很像，但最后不输出 10 类分数，而是输出一个固定长度的向量。

## 4. 文本编码器做什么

文本编码器负责：

```text
字符编号 [batch, text_len] -> 文本向量 [batch, embed_dim]
```

为了让初学者容易看懂，这里没有用复杂的 Transformer，而是使用字符级 embedding：

1. 每个汉字或字符先变成一个编号。
2. 每个编号查表得到一个小向量。
3. 对真实字符的向量求平均。
4. 再用线性层投影到和图片向量一样的维度。

这很简单，但足够演示“文本也能变成向量”。

## 5. 为什么要归一化向量

模型会对图片向量和文本向量做归一化：

```python
F.normalize(features, dim=-1)
```

归一化后，每个向量长度约等于 1。

这样两个向量的点积更像“方向相似度”，不会因为某个向量长度特别大就占便宜。

## 6. 相似度矩阵是什么

假设一个 batch 有 4 对图文：

```text
image_0 <-> text_0
image_1 <-> text_1
image_2 <-> text_2
image_3 <-> text_3
```

模型会算出一个 `4 x 4` 的相似度矩阵：

```text
          text_0  text_1  text_2  text_3
image_0     ?       ?       ?       ?
image_1     ?       ?       ?       ?
image_2     ?       ?       ?       ?
image_3     ?       ?       ?       ?
```

对角线位置是正确配对：

```text
image_0 对 text_0
image_1 对 text_1
image_2 对 text_2
image_3 对 text_3
```

训练目标就是让对角线分数最高。

## 7. 对比学习 loss 做了什么

对比学习不是只看一个正样本，而是在一个 batch 里同时制造很多负样本。

例如对 `image_0` 来说：

- `text_0` 是正样本。
- `text_1, text_2, text_3` 都是负样本。

模型既要学会“正确图文更接近”，也要学会“不匹配图文更远”。

本项目里同时训练两个方向：

```text
图片找文字：image -> text
文字找图片：text -> image
```

最后 loss 是两个方向的平均值。

## 8. logit_scale 是什么

`logit_scale` 是一个可学习的温度参数。

它会放大或缩小相似度分数：

```text
logits = scale * image_features @ text_features.T
```

如果分数太平，softmax 分不清谁更匹配。

如果分数太尖，训练可能不稳定。

所以我们让模型自己学习一个合适的 scale，并限制最大值，避免数值过大。

## 9. 这条多模态线和 LLM、视觉线的关系

三条学习线不是孤立的：

- LLM 线：学习“文本如何变 token，token 如何预测下一个 token”。
- 视觉线：学习“图片如何变张量，CNN 如何提取图像特征”。
- 多模态线：学习“图片向量和文本向量如何进入同一个语义空间”。

未来更大的视觉语言模型，会把这些思想组合起来：

```text
图片编码器 -> 图片特征
文本模型 -> 文本理解和生成
对齐模块 -> 把图片特征接入文本模型
```

所以当前的 TinyCLIP 是多模态学习的入门骨架。
