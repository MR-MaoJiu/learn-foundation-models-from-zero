# 视觉模型从 0 原理

这一部分用 MNIST 手写数字识别学习视觉模型。它不是为了追求最高分，而是为了让你看懂一条完整的视觉训练链路：

1. 图片如何变成张量。
2. 卷积神经网络如何提取图像特征。
3. 模型如何输出 0 到 9 的分类分数。
4. loss 如何告诉模型“你猜错了多少”。
5. 反向传播如何一点点改进模型参数。

## 1. 图片在模型眼里是什么

电脑里的图片通常是像素矩阵。

MNIST 图片是灰度图，每张大小是 `28 x 28`。灰度图只有一个颜色通道，所以一张图片可以表示成：

```text
[1, 28, 28]
```

含义是：

- `1`：通道数，灰度图只有 1 个通道。
- `28`：图片高度。
- `28`：图片宽度。

训练时不会一张一张喂给模型，而是一次喂一批，例如 batch size 是 64，那么输入形状就是：

```text
[64, 1, 28, 28]
```

含义是：一次给模型看 64 张图片。

## 2. 为什么不用普通全连接网络

如果把 `28 x 28` 图片直接拉平成 784 个数字，再送进全连接网络，也能训练。

但图片有一个重要特点：相邻像素之间有空间关系。例如数字 `8` 的上下两个圈，数字 `1` 的竖线，数字 `0` 的闭合轮廓，都依赖局部形状。

卷积神经网络 CNN 更适合图片，因为它会用一个小窗口在图片上滑动，寻找局部图案。

## 3. 卷积层做了什么

卷积层可以理解为很多个“小型图案探测器”。

例如：

- 有的卷积核可能学会检测竖线。
- 有的卷积核可能学会检测横线。
- 有的卷积核可能学会检测弯曲边缘。

刚开始这些卷积核是随机的。训练后，它们会逐渐变成对分类有用的图案探测器。

在代码里，卷积层是：

```python
nn.Conv2d(in_channels, out_channels, kernel_size=3, padding=1)
```

参数含义：

- `in_channels`：输入通道数。灰度图一开始是 1。
- `out_channels`：输出通道数，也可以理解为“学多少种图案探测器”。
- `kernel_size=3`：每次看 `3 x 3` 的小区域。
- `padding=1`：在图片边缘补一圈 0，让卷积后宽高不变。

## 4. 激活函数 ReLU 做了什么

卷积层本质上是线性计算。只堆线性层，模型表达能力会很弱。

ReLU 是一个非常常见的非线性函数：

```text
ReLU(x) = max(0, x)
```

它的作用是：

- 正数保留。
- 负数变成 0。

这会让模型可以表达更复杂的图案组合。

## 5. 池化层 MaxPool 做了什么

池化层会缩小图片特征图。

例如：

```python
nn.MaxPool2d(2)
```

会把宽高都缩小一半：

```text
28 x 28 -> 14 x 14
14 x 14 -> 7 x 7
```

它的直觉是：如果某个小区域里出现了重要特征，只保留最强的那个响应。

这样做有两个好处：

- 减少计算量。
- 让模型对轻微平移更稳定。

## 6. 分类头做了什么

卷积层提取的是图像特征，最后还需要把特征变成 10 个类别分数。

在这个项目里，分类头最后输出：

```text
[batch_size, 10]
```

每一行对应一张图片，10 个数字分别代表模型认为它是 `0, 1, 2, ..., 9` 的分数。

这些分数叫 `logits`。

注意：`logits` 不是概率。它们还没有经过 softmax。

## 7. 交叉熵 loss 做了什么

分类任务常用交叉熵：

```python
F.cross_entropy(logits, labels)
```

它会做两件事：

1. 把 logits 转成概率分布。
2. 惩罚正确类别概率太低的情况。

例子：

如果图片真实标签是 `7`，模型给数字 7 的分数很高，loss 就低。

如果模型给数字 3 的分数最高，loss 就高。

训练的目标就是让 loss 越来越低。

## 8. 准确率是什么

loss 适合训练，但人更容易理解准确率。

准确率计算方式：

```text
预测正确的图片数量 / 图片总数量
```

代码里会用：

```python
logits.argmax(dim=-1)
```

找到每张图片分数最高的类别，当作模型预测结果。

## 9. 训练循环发生了什么

视觉模型训练循环和 LLM 训练循环很像：

```text
读取一批图片和标签
-> 模型输出 10 个类别分数
-> 交叉熵计算 loss
-> loss.backward() 计算梯度
-> optimizer.step() 更新参数
-> 定期在验证集计算 loss 和准确率
```

你可以把训练理解成不断重复的纠错过程：

- 模型先乱猜。
- loss 告诉它错得多严重。
- 梯度告诉它参数应该往哪个方向改。
- 优化器真正修改参数。

## 10. 这条视觉线学完后你应该理解什么

跑通这一部分后，你应该能回答：

- 为什么图片输入是 `[batch, channels, height, width]`。
- CNN 为什么适合图片。
- 卷积层、ReLU、池化层、全连接层各自负责什么。
- logits、loss、accuracy 分别是什么。
- 训练集和验证集为什么要分开。

这为后面的多模态学习打基础。多模态里的图片编码器，本质上就是把视觉模型的“分类头”换成“向量输出头”。
