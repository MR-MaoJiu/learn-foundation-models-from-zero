语言模型的目标是根据已经出现的文字预测下一个文字。它不是先理解世界再回答问题，而是在大量文本中学习词语、句子和知识之间的统计关系。

训练一个小型语言模型可以分成四个步骤。第一步是准备语料，语料就是模型要阅读的文本。第二步是训练分词器，把文字切成 token。第三步是把 token 排成序列，构造输入和目标。第四步是让神经网络不断预测下一个 token，并根据错误调整参数。

神经网络中的参数可以理解为很多可以被调节的旋钮。训练开始时，这些旋钮几乎是随机的，所以模型输出很混乱。每一次训练都会计算损失，损失越大说明预测越差。反向传播会告诉每个参数应该往哪个方向调整。

Transformer 是现代语言模型最常用的结构。它的核心是注意力机制。注意力机制会让每个位置根据上下文选择要关注的其他位置。例如在句子“我把书放在桌子上，因为它很平”中，模型需要判断“它”更可能指代“桌子”。

因果注意力表示模型只能看见当前位置之前的 token，不能看见未来的答案。如果模型在训练时偷看未来，它在真实生成时就会失去能力。语言模型生成文本时，每次只生成一个 token，然后把这个 token 放回上下文，再继续生成下一个 token。

词表是 tokenizer 能识别的 token 集合。中文可以按字切分，也可以使用 BPE 这样的子词算法。BPE 会从字符开始，反复合并常见片段。例如“人工智能”可能被切成“人工”和“智能”，也可能被切成更细的片段。

Embedding 层会把 token id 转成向量。向量是一串数字，模型用这些数字表示词语的含义、语法和上下文作用。训练后，相似的词语往往会在向量空间中更接近。

位置编码告诉模型 token 的顺序。没有位置信息时，“猫追狗”和“狗追猫”会变得很难区分。RoPE 是一种常见的位置编码方法，它通过旋转向量的方式把相对位置信息加入注意力计算。

RMSNorm 是一种归一化方法。它让每一层的数值保持在比较稳定的范围内，避免训练过程中数值爆炸或消失。稳定的数值会让模型更容易学习。

SwiGLU 是一种常见的前馈网络结构。它先把隐藏向量扩展到更大的维度，再用门控机制控制信息通过，最后压回原来的维度。这个模块帮助模型记住和组合更复杂的模式。

训练语言模型时，batch size 表示一次看多少段文本。sequence length 表示每段文本有多少 token。显存消耗大致会随着 batch size、sequence length、层数和隐藏维度增加而增加。

学习率控制每次参数更新的步子大小。学习率太大，模型可能震荡甚至发散；学习率太小，训练会非常慢。常见做法是先 warmup，让学习率从小变大，然后再逐渐衰减。

过拟合表示模型只记住训练语料，却不能很好地处理新文本。如果训练损失下降，但验证损失变差，就可能发生了过拟合。增加数据、降低模型容量、使用 dropout 或提前停止都可能有帮助。

一个语言模型会包含大量可训练数字。每个参数如果用 float32 存储，需要四个字节。训练时还要存梯度、优化器状态和中间激活，所以实际显存需求会比模型文件本身高很多。

从零训练模型和微调模型不同。从零训练时模型一开始什么都不会，需要大量语料和算力。微调是在已有模型基础上继续训练，成本低很多，也更适合大多数应用。

评估语言模型可以看 loss，也可以看困惑度 perplexity。困惑度越低，说明模型对下一个 token 的不确定性越小。不过困惑度不能完全代表模型是否有用，还需要人工检查生成质量。

生成文本时可以使用 temperature 控制随机性。temperature 越低，模型越保守；temperature 越高，模型越发散。top-k 和 top-p 采样会限制候选 token，让生成结果更稳定。

如果你是初学者，最重要的不是一开始就训练出强大的模型，而是亲手跑通完整流程。理解每个文件、每个函数和每个张量的形状，比盲目追求参数量更重要。

今天我们从小模型开始。先让模型学会在一小段中文材料上降低 loss，再逐步增加数据、层数、隐藏维度和上下文长度。每一步都观察日志，确认模型真的在学习。
